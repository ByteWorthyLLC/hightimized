State machine (RESEARCH.md Q11 lines 962-996):

AuditStatus:
  | { phase: 'idle' }
  | { phase: 'ocr-loading'; progress: number }
  | { phase: 'parsing' }
  | { phase: 'audited'; lineItems: AuditedLine[]; flaggedCount: number }
  | { phase: 'letter-ready' }

AuditedLine: LineItem augmented with `flag: FlagResult | null` and `chargemasterRow: ChargemasterRow | null`.

Actions:
  FILE_ACCEPTED, OCR_PROGRESS, OCR_COMPLETE, AUDIT_COMPLETE, OCR_ERROR, LETTER_GENERATING, LETTER_COMPLETE, RESET

Errors are managed in a separate useState<{ title: string; body: string } | null>(null) — does not change reducer state per RESEARCH.md Q11 line 997.

Expanded state for trace cards is local useState in App: `Record<string, boolean>` keyed by cptCode.

6. Defines the AuditStatus discriminated union and AuditAction union per RESEARCH.md Q11 lines 962-980.

7. Implements `auditReducer(state, action): AuditStatus` per RESEARCH.md lines 981-995 with the addition of:
   - LETTER_GENERATING: returns a phase: 'letter-generating' (extend the union to include this) carrying the audited lineItems forward
   - LETTER_COMPLETE: returns to 'audited' with the original lineItems (the reducer treats letter-generation as a transient state that returns to audited)
   - OCR_ERROR: returns { phase: 'idle' } (treats the error as a reset of the pipeline). The action carries `message: string`; the reducer ignores the message but ToastError surfaces it via the separate `error` useState. Including the action in the union keeps the AuditAction type exhaustive — handleFileAccepted's catch block must dispatch OCR_ERROR (not RESET) so the action history is auditable.
   - RESET: returns { phase: 'idle' }

8. Implements the `App` component:

   - `const [status, dispatch] = useReducer(auditReducer, { phase: 'idle' })`
   - `const [error, setError] = useState<{ title: string; body: string } | null>(null)`
   - `const [expanded, setExpanded] = useState<Record<string, boolean>>({})`
   - `const [letterLoading, setLetterLoading] = useState(false)`

   - `handleFileAccepted = useCallback(async (file: File) => { ... })`:
     - dispatch FILE_ACCEPTED.
     - try block:
       - text = await ocrFirstPageOfPdf(file, (p) => dispatch({ type: 'OCR_PROGRESS', progress: p }))
       - dispatch OCR_COMPLETE
       - lineItems = parseBillText(text)
       - db = await getChargemasterDb()
       - audited = lineItems.map((item) => { const row = lookupChargemaster(db, item.cptCode); const flag = row ? flagLine(item.charge, row) : null; return { ...item, flag, chargemasterRow: row } })
       - dispatch AUDIT_COMPLETE with the audited array
     - catch err:
       - if err is the OCR-specific error: setError({ title: "Couldn't read that bill.", body: "The text wasn't clear enough. Try a higher-quality scan." })
       - else if it's a sql.js init/fetch error: setError({ title: "Data failed to load.", body: "Reload the page to try again." })
       - else default: setError({ title: "Something went wrong.", body: "Reload the page to try again." })
       - dispatch `{ type: 'OCR_ERROR', message: err instanceof Error ? err.message : String(err) }` so the DropZone returns to idle. The reducer maps OCR_ERROR back to `{ phase: 'idle' }` (same effect as RESET); ToastError displays the body copy set above. Using OCR_ERROR (not RESET) keeps the AuditAction union exhaustive — every action declared in the union must be dispatched somewhere, otherwise the type is dead code.

   - `handleFileRejected = useCallback((reason: 'wrong-type' | 'too-large' | 'unknown') => { ... })`:
     - if reason === 'wrong-type': setError({ title: 'File not supported.', body: "Drop a PDF — image files work in a future version." })
     - if reason === 'too-large': setError({ title: 'File too large.', body: 'Drop a smaller PDF (under 50MB).' })
     - else: setError({ title: 'Could not accept that file.', body: 'Try again with a hospital bill PDF.' })

   - `handleToggleExpand = useCallback((cptCode: string) => setExpanded((prev) => ({ ...prev, [cptCode]: !prev[cptCode] })), [])`

   - `handleGenerate = useCallback(async () => { ... })`:
     - if status.phase !== 'audited' return.
     - setLetterLoading(true).
     - try:
       - const flagged = status.lineItems.filter((l) => l.flag).map((l) => ({
           cptCode: l.cptCode,
           description: l.description,
           charged: l.flag!.charged,
           publishedRate: l.flag!.comparisonRate,
           multiplier: l.flag!.multiplier,
         }))
       - const bytes = await generateDisputeLetter(flagged)
       - Use Blob ([bytes], { type: 'application/pdf' })
       - const url = URL.createObjectURL(blob)
       - const a = document.createElement('a')
       - a.href = url
       - a.download = `hightimized-dispute-${new Date().toISOString().slice(0, 10)}.pdf`
       - a.click()
       - setTimeout(() => URL.revokeObjectURL(url), 30000)
     - catch:
       - setError({ title: "Letter didn't generate.", body: "Reload the page and try again. Your bill data is still here." })
     - finally:
       - setLetterLoading(false)

   - DropZone state derivation:
     - 'idle' when status.phase is 'idle'
     - 'processing' when status.phase is 'ocr-loading' or 'parsing'
     - 'complete' when status.phase is 'audited' or 'letter-generating'
     - (DropZone handles 'dragging' internally via react-dropzone)

   - Render structure (UI-SPEC.md lines 416-444):
     - <PrivacyBanner />
     - <main className={styles.content}>
       - <section className={styles.heroZone}><DropZone state={dropZoneState} ... /></section>
       - {status.phase === 'audited' || status.phase === 'letter-generating' ? (
           <section className={styles.list} aria-live="polite">
             {status.lineItems.map((line) => (
               <LineItemCard
                 key={line.cptCode}
                 cptCode={line.cptCode}
                 description={line.description}
                 charge={line.charge}
                 flag={line.flag}
                 isExpanded={!!expanded[line.cptCode]}
                 onToggleExpand={() => handleToggleExpand(line.cptCode)}
               />
             ))}
           </section>
         ) : null}
     - </main>
     - {(status.phase === 'audited' || status.phase === 'letter-generating') && status.lineItems.length > 0 ? (
         <GenerateLetterButton
           flaggedCount={status.lineItems.filter((l) => l.flag).length}
           isLoading={letterLoading}
           onGenerate={handleGenerate}
         />
       ) : null}
     - {error && <ToastError title={error.title} body={error.body} onDismiss={() => setError(null)} />}
