# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Vertical Slice.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-built primitives in `src/components/` |
| Icon library | none — CSS-only chevron (caret) via border trick; right-arrow glyph is Unicode `→` (U+2192) |
| Font | System stack (see Typography) — zero CDN calls, zero deps |

**Rationale (from CONTEXT.md):** Vanilla CSS modules. No Tailwind, no shadcn, no Material UI. Minimal dependency footprint for Phase 1. The existing `src/index.css` CSS custom properties are the token source.

---

## Spacing Scale

All values are multiples of 4. Token names map to CSS custom properties in `src/index.css`.

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--space-xs` | 4px | Icon gaps, badge inner padding (vertical) |
| sm | `--space-sm` | 8px | Badge inner padding (horizontal), list-item gap between CPT code and description |
| md | `--space-md` | 16px | Card internal padding, DropZone padding top/bottom, input margin |
| lg | `--space-lg` | 24px | Section gap between DropZone and LineItemList, card row gap |
| xl | `--space-xl` | 32px | Hero section top padding, sticky footer height |
| 2xl | `--space-2xl` | 48px | Hero section bottom padding |
| 3xl | `--space-3xl` | 64px | Page top and bottom padding on desktop |

**Exceptions:**
- Touch targets: minimum 44px height on all interactive elements (WCAG 2.5.8 AA). The GenerateLetterButton height is 52px to exceed this threshold.
- DropZone minimum height: 180px on mobile, 240px on desktop — larger than the spacing scale to feel like a destination, not a button.
- PrivacyBanner height: 40px — fixed; vertically centers single-line copy.

## Typography

**Font stack (system, zero deps):**
```
--sans: system-ui, 'Segoe UI', Roboto, Helvetica Neue, Arial, sans-serif;
--mono: ui-monospace, 'Cascadia Code', 'Source Code Pro', Menlo, Consolas, monospace;
--heading: system-ui, 'Segoe UI', Roboto, Helvetica Neue, Arial, sans-serif;
```
These are already declared in `src/index.css`. No new font variables needed for Phase 1.

| Role | Size | Weight | Line Height | CSS Class / Usage |
|------|------|--------|-------------|-------------------|
| Display | 28px | 700 | 1.2 | DropZone heading: "Drop your hospital bill here." |
| Heading | 20px | 700 | 1.3 | Section label ("Your bill" once items are loaded), ToastError title |
| Body | 16px | 400 | 1.5 | LineItemCard description text, TraceDetail rows, PrivacyBanner |
| Label | 14px | 400 | 1.4 | Subtitle under Display heading (descriptive, low-emphasis) |

**Two-weight rule (locked):**
- **400** = body, descriptions, trace rows, privacy copy, low-emphasis labels.
- **700** = headings, CPT codes, currency amounts, FlagBadge text, button labels, the bolded multiplier word in TraceDetail.
- No 500 / no 600 anywhere. The contrast between 400 and 700 is the primary typographic signal — appropriate for a bill-audit UI where numerical accuracy matters more than typographic subtlety.

**Mono usage:**
- CPT codes: `--mono` 14px weight 700 `--text-h`.
- Currency amounts (`$750.00`): `--mono` 14px weight 700 `--text-h`.
- FlagBadge multiplier (`4.0×`): `--mono` 14px weight 700.

**Other rules:**
- No italic anywhere in Phase 1.
- No font sizes outside this scale — do not interpolate.

### 2. DropZone

**File:** `src/components/DropZone/DropZone.tsx`

**Visual (idle state):**
- Rounded rectangle, `border-radius: 12px`.
- Border: `2px dashed var(--border)`.
- Background: `var(--bg)`.
- Centered content column: Display heading + subtitle + (implicit click affordance).
- Min-height: 180px mobile, 240px desktop.
- Max-width: matches page content column (100% mobile, 672px desktop — `max-w-2xl` equivalent: `max-width: 672px`).

**Visual (dragging state):**
- Border color: `var(--accent)` (solid, not dashed: `2px solid var(--accent)`).
- Background: `var(--accent-bg)`.
- Transition: `border-color 150ms ease, background-color 150ms ease`.

**Visual (processing state — OCR running):**
- Border returns to idle style.
- Heading replaced by spinner + loading copy (see Copy section).
- Spinner: 24px CSS-only `border` spinner, `border-top-color: var(--accent)`, `animation: spin 0.8s linear infinite`.

**Visual (complete state — items loaded):**
- DropZone collapses to a 48px compact bar: `"Drop another bill to start over."` at body size, centered.
- Border becomes solid 1px `var(--border)`, background `var(--surface)`.

**States:**
1. `idle` — default
2. `dragging` — file hovering over zone
3. `processing` — OCR running (file accepted, processing underway)
4. `complete` — line items rendered below; drop zone collapses

**Interactions:**
- Drag-and-drop via `react-dropzone` v14.
- Click anywhere in the zone opens native file picker (fallback `<input type="file" accept=".pdf">`).
- Keyboard: zone is focusable (`tabIndex={0}`), `Enter` or `Space` opens file picker.

**ARIA:**
```
role="button"
aria-label="Drop your hospital bill here, or press Enter to browse"
aria-describedby="dropzone-subtitle"
tabIndex={0}
```
The hidden file input has `aria-hidden="true"`.

**Accepted types:** `.pdf` only (Phase 1). `accept={{ 'application/pdf': ['.pdf'] }}`.

**Error trigger:** Wrong file type → renders ToastError (see below). DropZone stays in idle state.

**Dimensions:**
- Mobile: width 100%, min-height 180px
- Desktop: max-width 672px, min-height 240px, centered via `margin: 0 auto`

**Visual (unflagged):**
- White card (`--bg`), `border: 1px solid var(--border)`, `border-radius: 8px`.
- Padding: `var(--space-md)` (16px) all sides.
- Three-column layout (mobile: stacked two rows):
  - Left: CPT code in `--mono` 14px weight 700 `--text-h`
  - Center: description text 16px weight 400 `--text-h` (LineItemCard background is `--surface`; using `--text-h` for AA contrast — see Color § contrast ratios)
  - Right: charge amount in `--mono` 14px weight 700 `--text-h`, right-aligned

**Visual (flagged):**
- Same card structure.
- FlagBadge appears inline after description (see FlagBadge below).
- Left border accent: `border-left: 3px solid var(--flag-red)`.
- Background: `#fff9f9` light mode / `#1f1515` dark mode — subtle blush tint to make flagged rows scannable.

**States:**
1. `normal` — no flag

3. `expanded` — trace detail visible below

**Interactions:**

**ARIA:**

tabIndex={0}

**Dimensions:**

### 5. TraceDetail

**File:** `src/components/TraceDetail/TraceDetail.tsx`

**Visual:**
- Revealed below the LineItemCard row when expanded.
- Background: `var(--surface)` (`--code-bg` equivalent: `#f4f3ec`), `border-radius: 0 0 8px 8px`, `border-top: 1px solid var(--border)`.
- Padding: `var(--space-md)` (16px).
- Four data rows in monospace 14px, one per line:
  1. `Rule: charge > 1.5× hospital published rate (matched).`
  2. `Charged: $750.00`
  3. `Hospital published rate: $187.50`
  4. `Multiplier: 4.0×`
- Below the four rows, a plain-English sentence in body text 16px weight 400 `--text`:
  `"We found you were charged FOUR times what this hospital publicly says they charge for this code."`
  - The word "FOUR" (or the spelled-out multiplier) renders in weight 700 `--text-h`.

**States:**
1. `hidden` — `display: none` (not `visibility: hidden` — remove from layout entirely)
2. `visible` — rendered

**Interactions:** Controlled by parent LineItemCard. No direct interactions.

**ARIA:**
```
id={`trace-${cptCode}`}
role="region"
aria-label={`Rule trace for CPT ${cptCode}`}
```
Hidden state uses `hidden` attribute (not CSS `display: none`) to correctly hide from assistive technology.

**Motion (expand/collapse):** CSS `max-height` transition:
```css
max-height: 0;
overflow: hidden;
transition: max-height 250ms ease-in-out;
```
Expanded: `max-height: 300px` (sufficient for 4 data rows + plain-English sentence). This avoids JS-measured height and works with CSS-only transitions.

**Dimensions:** width 100%, height auto (determined by content within max-height).

### 6. GenerateLetterButton (sticky footer)

**File:** `src/components/GenerateLetterButton/GenerateLetterButton.tsx`

**Visual (active state — flagged lines exist):**
- Sticky footer: `position: sticky; bottom: 0; z-index: 50`.
- Footer container: full-width, background `var(--bg)`, `border-top: 1px solid var(--border)`, padding `var(--space-md)` (16px).
- Button: `background: var(--accent)`, `color: #fff`, `border: none`, `border-radius: 8px`.
- Height: 52px. Full-width on mobile. `max-width: 480px`, centered on desktop.
- Text: 16px weight 700, `"Generate dispute letter →"`.

**Visual (disabled state — zero flagged lines):**
- Button: `background: var(--surface)`, `color: var(--text)`, `cursor: not-allowed`, `opacity: 0.6`.
- Text: `"No overcharges to dispute."` (no arrow glyph in disabled state).

**Visual (loading state — PDF generating):**
- Button disabled, text replaced by `"Building your letter…"`.
- 16px CSS spinner (same spinner style as DropZone processing) appears left of text.

**Visual (footer not rendered):**
- Footer is not rendered at all until at least one LineItemCard exists (i.e., until OCR completes and line items are present). Before that, no sticky footer is visible.

**States:**
1. `hidden` — no line items loaded yet; footer not mounted
2. `disabled` — line items loaded, zero flags
3. `active` — one or more flagged lines exist
4. `loading` — PDF generation in progress

**Interactions:**
- Active state: click triggers pdf-lib PDF generation + `URL.createObjectURL` download.
- Keyboard: standard `<button>` focus + `Enter`/`Space` activation.
- Object URL is revoked after 30 seconds (cleanup side effect).

**ARIA:**
```
aria-disabled={isDisabled}
aria-label={isDisabled ? "No overcharges found to dispute" : "Generate dispute letter PDF and download"}
```
Use `aria-disabled` (not `disabled`) to keep the element focusable and allow screen readers to read the disabled copy.

**Dimensions:** footer height `var(--space-xl)` (32px padding) + 52px button = ~84px total footer zone.

### 7. ToastError

**File:** `src/components/ToastError/ToastError.tsx`

**Visual:**
- Fixed bottom-right corner on desktop, bottom-center on mobile.
- `position: fixed; bottom: var(--space-lg); right: var(--space-lg)` (desktop).
- `position: fixed; bottom: calc(84px + var(--space-md)); left: 50%; transform: translateX(-50%)` (mobile — sits above sticky footer).
- Background: `#1c1917` (near-black, readable in both modes). `color: #fafaf9`. `border-radius: 8px`.
- Width: `max-width: 360px`, `min-width: 240px`.
- Padding: `var(--space-md)` (16px).
- Title: 14px weight 700 `#fafaf9`. Body: 14px weight 400 `#d6d3d1`.
- Auto-dismisses after 5 seconds. `×` close button top-right: 20×20px, `color: #a8a29e`, tap target padded to 32×32px.

**States:**
1. `hidden` — not mounted
2. `visible` — mounted, countdown running
3. `dismissing` — fade-out transition (150ms)

**Interactions:**
- Auto-dismiss at 5 seconds.
- Manual dismiss via `×` button.
- Does not prevent interaction with the rest of the page.

**ARIA:**
```
role="alert"
aria-live="assertive"
aria-atomic="true"
```
This ensures screen readers announce the error immediately on mount.

**Motion:** `animation: toastIn 200ms ease-out` on mount (slide up 8px + fade in). `animation: toastOut 150ms ease-in` on dismiss (fade out, no movement). Prefer `@media (prefers-reduced-motion: reduce)` to suppress both animations.

**Dimensions:** max-width 360px, min-height 60px.

### Stub PDF Letter

The generated PDF contains credible structure (not lorem ipsum):

```
[YOUR NAME]
[YOUR ADDRESS]
[CITY, STATE ZIP]

{YYYY-MM-DD}

Billing Department
Memorial Regional Medical Center
[HOSPITAL ADDRESS]

RE: Dispute of Charges — Itemized Bill Dated {YYYY-MM-DD}

Dear Billing Department,

I am writing to dispute the charge for CPT code 99213 (Office visit, est. patient, 
low complexity) on my recent statement.

My bill shows a charge of $750.00 for this service. Memorial Regional Medical Center's 
own publicly-posted chargemaster lists the rate for CPT 99213 at $187.50 — a difference 
of $562.50, or 4.0 times the published rate.

Under 45 CFR §180 (Hospital Price Transparency Final Rule), hospitals are required to 
publish their standard charges for all items and services. I am requesting an immediate 
review and correction of this line item to reflect the published rate of $187.50, or a 
written explanation of the billing discrepancy within 30 days.

This letter is sent in good faith. I expect a written response within 30 days of receipt.

Sincerely,

_____________________________
[YOUR NAME]
[YOUR PHONE / EMAIL]
```

**Note:** `[YOUR NAME]`, `[YOUR ADDRESS]` etc. are literal bracket placeholders. The user edits them before printing. Phase 3 adds form-field input for these values.

PDF filename: `hightimized-dispute-{YYYY-MM-DD}.pdf` (date from `new Date().toISOString().slice(0,10)`).

## Motion

All motion uses CSS only. No Framer Motion, no GSAP, no JS animation libraries in Phase 1.

```css
/* Respect reduced motion — wrap all keyframe animations */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

| Component | Trigger | Property | Duration | Easing |
|-----------|---------|----------|----------|--------|
| DropZone | File drag enters | `border-color`, `background-color` | 150ms | `ease` |
| DropZone | File drag leaves | `border-color`, `background-color` | 150ms | `ease` |
| FlagBadge | Mount (after OCR) | `opacity` 0→1, `transform: scale(0.85→1)` | 200ms | `ease-out` |
| TraceDetail | Expand (click card) | `max-height` 0→300px | 250ms | `ease-in-out` |
| TraceDetail caret | Expand | `transform: rotate(0deg→90deg)` | 200ms | `ease-in-out` |
| TraceDetail | Collapse | `max-height` 300px→0 | 200ms | `ease-in-out` |
| GenerateLetterButton | Hover (active state) | `filter: brightness(1.08)` | 120ms | `ease` |
| GenerateLetterButton | Active (mousedown) | `transform: scale(0.98)` | 80ms | `ease` |
| ToastError | Mount | `opacity` 0→1, `translateY` 8px→0 | 200ms | `ease-out` |
| ToastError | Dismiss | `opacity` 1→0 | 150ms | `ease-in` |
| OCR spinner | Continuous | `transform: rotate(360deg)` | 800ms | `linear infinite` |

**No layout animations** (no animating `height`, `width`, or `top`/`left` except via `transform`). Only `max-height` is animated directly (acceptable trade-off for TraceDetail — content height is bounded and predictable).
